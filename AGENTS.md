# start-code プロジェクト概要

## プロジェクトの目的
これからプログラミングを始める方が、素早く開発・実行環境を構築するためのセットアップスクリプト集です。
1台の開発マシンに複数バージョンの環境をインストールし、アプリケーションごとに切り替えて使用できるようにすることを目的としています。各言語において標準的なバージョン管理ツールやパッケージ管理ツールのセットアップを自動化します。

## 対象プラットフォーム
- **macOS** (動作確認済み): x86_64 / ARM64, macOS 14 Sonoma以降, zsh
- **Windows** (開発進行中): PowerShell（Scoop等を利用したスクリプト群を開発中）

## 対象プログラミング言語と対応ツール
| 言語 | バージョン管理 | パッケージ管理 |
|:---|:---|:---|
| **Ansible** | venv | ansible-galaxy |
| **JavaScript** | nvm | npm |
| **Python** | pyenv | venv+pip, Pipenv, Poetry |

## ディレクトリ・ファイル構成

- **`README.md` / `README_en.md`**
  プロジェクトの全体像、システム要件、利用可能なスクリプトとその実行方法を詳しく記載したユーザ向けドキュメント。

- **`macos/`**
  macOS向けの環境構築シェルスクリプト群。
  - **`install/`**: ユーザが直接実行するエントリーポイント (`ansible.sh`, `javascript-node.sh`, `python.sh` など)。
  - **`lib/`**: `install/` 配下のスクリプトから呼び出される共通処理やインストールロジックをまとめたライブラリ (`constants.sh`, `functions.sh`, `homebrew.sh`, `nvm.sh`, `pyenv.sh` など)。

- **`windows/`** （開発中）
  Windows向けの環境構築PowerShellスクリプト群。
  - **`lib/`**: PowerShellモジュールおよびそのテストコード群 (`Constants.psm1`, `Functions.psm1`, `Scoop.psm1`, `*.Tests.ps1`)。現在はScoop等を利用した基盤機能の実装が進められています。
  - 今後、macOS版と同等の機能を提供する `install/` ディレクトリ内のスクリプトが追加されていく予定です。

- **`docs/`**
  各種ツールの詳細な使用方法に関する補足ドキュメント (`ansible.md` など)。

- **`.github/ workflows/`**
  GitHub Actionsのワークフロー定義ファイル群。対象環境でのCIテスト（インストール成功の検証など）に用いられています。

## AIエージェントへのガイドライン
- 本プロジェクトは、異なるOS・環境向けに一貫したユーザー体験（セットアップの容易さ）を提供することを目指しています。
- シェルスクリプトは主に `zsh` コンパチブルで書かれています。
- 現在、Windows向けサポート（PowerShell）の機能拡張がアクティブに行われています。Windows向けのスクリプトを実装する際は、既存の `macos` 用スクリプトの処理フローや設計パターン（`install` と `lib` の分離など）を参考に一貫性を持たせてください。
